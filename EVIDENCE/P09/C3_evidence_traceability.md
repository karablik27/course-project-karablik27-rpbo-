## Evidence & Traceability (C3 — ★★)

Все артефакты генерации SBOM и SCA сохраняются в директорию:

```text
EVIDENCE/P09/
 ├── sbom.json          # CycloneDX SBOM, полный перечень пакетов
 ├── sca_report.json    # Отчёт Grype со всеми CVE
 └── sca_summary.md     # Человеко-читаемая сводка по уровням severity
```

---

### Использование артефактов (для DS1 и итогового отчёта)

Эти файлы могут использоваться в дальнейшем:

**SBOM (`sbom.json`)**
- Можно открыть в любой CycloneDX-совместимой системе (например, Dependency-Track, OWASP-инструментах).
- Используется для:
  - анализа состава ПО;
  - сравнения зависимостей между релизами;
  - подготовки DS-раздела итогового отчёта (описание компонентов приложения и динамики обновления зависимостей).

**SCA Report (`sca_report.json`)**
- Содержит все найденные CVE, их severity, affected versions и статусы фиксов.
- Применяется для:
  - анализа регрессий по безопасности;
  - проверки, появились ли новые уязвимости после обновления зависимостей;
  - трассировки решений по remediation (какие версии/пакеты были обновлены).

**SCA Summary (`sca_summary.md`)**
- Итоговая агрегация по severity (Critical / High / Medium / Low).
- Используется как:
  - краткая справка в DS1-разделе;
  - человекочитаемый отчёт для code review / security review;
  - ссылка в описании релиза или PR, подтверждающая текущее состояние безопасности.

---

### Привязка к CI

В каждом прогоне CI артефакты автоматически генерируются и загружаются в job:

> **Security - SBOM & SCA**

Для текущего PR:
- job **успешно выполняется**;
- артефакты `sbom.json`, `sca_report.json`, `sca_summary.md` доступны для скачивания из интерфейса CI.

Это обеспечивает **полную воспроизводимость**:

- каждому коммиту в ветке соответствует свой уникальный набор артефактов;
- можно восстановить состояние зависимостей и результатов SCA на момент любого прогона CI;
- DS1-раздел итогового отчёта может опираться на конкретные версии SBOM/SCA, а не на «текущее» состояние репозитория.
